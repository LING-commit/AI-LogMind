### 2.1 插件加载与解析功能

#### TC-PLUG-001: 插件目录不存在时 daemon 正常启动
- **前置**: 修改 `plugins_dir` 为不存在的路径
- **步骤**: 启动 daemon
- **预期**: daemon 输出警告日志但仍正常进入 RUNNING 状态
- **结果**: ✅ 通过
- **备注**: 已修复（原实现将空目录视为启动失败）

#### TC-PLUG-002: 空插件目录不影响 daemon 运行
- **前置**: 创建空目录 `/tmp/empty-plugins`
- **步骤**: daemon 使用 `plugins_dir: /tmp/empty-plugins` 启动
- **预期**: 启动日志显示 "0 collectors, 0 analyzers, 0 alerts"，daemon 正常运行
- **结果**: ✅ 通过
- **备注**: 实际输出：`No plugins loaded, continuing without plugins` 后正常注册 D-Bus 服务

#### TC-PLUG-003: `ListPlugins` D-Bus 方法返回正确
- **前置**: 无插件加载
- **步骤**: 调用 D-Bus `ListPlugins`
- **预期**: 返回空 JSON 数组 `[]`
- **结果**: ✅ 通过
- **备注**: 返回 JSON 字符串 `s "[]"`

#### TC-PLUG-004: `GetPluginInfo` 返回未加载插件信息
- **前置**: 无插件加载
- **步骤**: 调用 `GetPluginInfo("test")`
- **预期**: 返回 `{"error": "Plugin not found"}`
- **结果**: ✅ 通过

#### TC-PLUG-005: `ReloadPlugins` 方法功能验证
- **前置**: daemon 正常运行
- **步骤**: 调用 `ReloadPlugins`
- **预期**: 返回 true/false；插 SDK 加载指定目录
- **结果**: ⚠️ 部分通过
- **问题**: #P1 – `ReloadPlugins` 使用**硬编码路径** `/usr/lib/logmind/plugins` 而非配置文件中 `plugins_dir`
- **建议**: 修改为使用 `m_config.plugins_dir()`
- **涉及文件**: `src/daemon/dbus_service.cpp:176`

#### TC-PLUG-006: 插件加载接口 ABI 兼容性检测
- **前置**: 无实际 `.so` 插件
- **步骤**: 代码审查 `plugin_loader.cpp` 中 `load_plugin` 方法
- **预期**: 调用 `plugin_abi_version()` 进行版本校验
- **结果**: ✅ 通过（代码审查）
- **备注**: `load_plugin` 使用 `dlsym("plugin_abi_version")` 获取 ABI 版本并校验

#### TC-PLUG-007: 接口虚函数表正确发射（代码审查）
- **前置**: 无
- **步骤**: 审查 `plugin_loader.cpp` 中 `dynamic_cast` 用法
- **结果**: ❌ 编译失败
- **问题**: #P0 – `ICollector`/`IAnalyzer`/`IAlert` 缺少关键函数（key function），`dynamic_cast` 导致 "undefined reference to typeinfo" 链接错误
- **修复状态**: ✅ 已修复
- **方案**: `src/daemon/plugin_api.cpp` 提供各抽象类的析构函数和 `parse_file` 默认实现

#### TC-PLUG-008~012: 插件生命周期钩子
- **状态**: ⏳ 阻塞
- **原因**: 尚无实际 `.so` 插件可供加载。需在 Phase 1 构造首个示例插件（如 `parser_nginx.so`）后方可验证 `on_load`/`on_unload`/`on_reload_config`。

#### 问题汇总

| ID | 严重度 | 状态 | 描述 | 建议 |
|----|--------|------|------|------|
| #P0-01 | P0 | ✅ 已修复 | 抽象接口类因缺少 key function 导致 `dynamic_cast` 链接失败 | 提供 `plugin_api.cpp` 给出析构函数定义 |
| #P1-01 | P1 | ❌ 待修复 | `ReloadPlugins` 硬编码路径 | 改用 `m_config.plugins_dir()` |
| #P2-01 | P2 | ⏳ 待验证 | `load_plugin` 共享库卸载时序 | 需待 Phase 1 插件开发完成后验证 `dlclose` 行为 |
